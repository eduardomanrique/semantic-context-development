
# Semantic Context Development

Code is implementation. Semantic Context is truth.

This repository explores **Semantic Context Development**, a software development model in which the primary source of truth is not code, but a living **Semantic Context** continuously maintained by AI agents and validated by humans through interaction.

The goal is to study whether a continuously evolving semantic source can preserve system intent better than code alone, support safer changes, answer questions about business rules and flows, and eventually help reimplement systems from meaning rather than from implementation details.

## Current scope

This repository currently focuses on:

- defining the concept and framing
- designing the structure of the Semantic Context
- defining the development workflow around AI agents
- creating a demo project to validate the idea
- exploring possible reverse engineering for legacy systems

## Initial repository structure

- `00-thesis.md` — framing and core thesis
- `AGENTS.md` — instructions for AI agents working in this repository
- `semantic-context/` — the living semantic source
- `notes/` — brainstorms and working notes
- `demo/` — demo project used to validate the model
